# CS-320 – Software Testing, Automation, and Quality Assurance

This repository segment contains academic artifacts developed for CS-320, with emphasis on systematic verification, validation, and automated testing practices in Java.

The course concentrated on constructing defensible unit tests, aligning implementation behavior with formal requirements, and evaluating test sufficiency through structural coverage analysis.

## Technical Focus Areas

- JUnit 5 test development
- Requirement-driven test case design
- Boundary value and negative testing strategies
- Exception validation and input constraint enforcement
- Code coverage analysis and adequacy evaluation
- Verification vs. validation methodology

## Projects Included

### Project One – Contact Service
Implementation of a constrained data model and service layer with comprehensive unit tests validating:
- Input length constraints
- Null handling safeguards
- Controlled state mutation
- Defensive exception handling

### Project Two – Testing Analysis and Reflection
Formal written evaluation of testing effectiveness, requirement alignment, and coverage sufficiency, supported by coverage metrics and reflective quality analysis.

## Competencies Demonstrated

- Test-driven validation discipline
- Structured risk mitigation through automated checks
- Alignment between specification and executable behavior
- Analytical reasoning regarding coverage adequacy

This course artifact reflects disciplined application of software quality principles within an academic development environment.
